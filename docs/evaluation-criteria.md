# 평가 기준 · Sparta_AX_Project

> 출처: `docs/references/AX 해커톤 평가 기준.md`, `docs/references/발제.md` / 확인일: 2026-07-01 / 적용 범위: 프로젝트 전 과정의 자기점검 기준

## 0. 전체 평가 관점

- 커리어개발: 40점
- AI테크: 60점
- 공통 핵심: 문제를 자기 말로 재정의하고, 구조화된 결과물과 실제 동작하는 완성도를 보여준다.

## 1. 커리어개발 40점

### A1 · 문제 정의

- 평가 내용: 문제를 자기 말로 재정의하고 접근 근거를 설명하는가.
- 적용: “이메일을 많이 만든다”가 아니라 “박람회 후 48시간 골든타임 안에 직책별 맥락과 상담 내용을 살려 후속 전환 가능성을 높이는 자동화”로 문제를 재정의한다.

### A2 · 구조화

- 평가 내용: 결과물과 발표자료의 전달력, 가독성, 구조화.
- 적용: Basic, Standard, Challenge 산출물을 구분하고, 평가자가 `email_drafts`, 파이프라인, 캠페인 기획안, 검수 리포트를 쉽게 확인하게 한다.

### A3 · 주도성

- 평가 내용: 요구를 넘어 실무 디테일을 주도적으로 고민·제안했는가.
- 적용: 이메일 결측, 상담 메모 결측·모호, 중복 리드, 수신 동의, Human-in-the-loop, 우선순위별 CTA 강도까지 설계한다.

## 2. AI테크 60점

### B1 · 문제 정의

- 평가 내용: 기술 관점에서 문제와 접근 근거를 설명하는가.
- 적용: CSV 입력을 분석하고, 도메인 컨텍스트와 직책별 규칙을 결합해 재현 가능한 생성 파이프라인으로 만든다.

### B2 · 기술적 문제해결

- 평가 내용: 막힌 지점과 기술 난관을 타당하게 풀었는가.
- 적용: 데이터 품질 이슈, review_flag, 중복 처리, 본문 길이, CSV/XLSX 형식, 캠페인 세그먼트 도출을 검증 가능한 절차로 푼다.

### B3 · 결과물 완성도·동작

- 평가 내용: Basic → Standard → Challenge 중 어디까지 동작하는가.
- 적용: 목표는 Basic, Standard, Challenge 전 요구사항 충족이다. Challenge 이상의 완성도를 위해 검수 리포트와 제출용 요약을 추가한다.

## 3. 발제 기준 세부 채점표

### Basic · 100점

| 항목 | 배점 | 적용 기준 |
|---|---:|---|
| 직책별 차별화 정확도 | 40 | 원장·팀장·사업개발의 소구점, CTA, 제목이 다름 |
| 개인화 완성도 | 20 | 맞춤 인사, 상담 내용 언급, 제품 연결, CTA 포함 |
| 본문 품질·예외 처리 | 30 | 250~400자, 자연스러운 한국어, review_flag, 이메일 결측 처리 |
| 제출 형식 | 10 | `email_drafts.csv/.xlsx`, 컬럼 `lead_id, subject, body, review_flag` |

### Standard · +30점

| 항목 | 배점 | 적용 기준 |
|---|---:|---|
| 파이프라인 재현성 | 15 | 새 CSV에도 같은 절차로 산출 가능 |
| 후속 운영 설계 | 15 | 소구점, CTA, 우선순위별 강도, review_flag 기준의 근거와 파이프라인 생성 이메일 샘플 1부가 있음 |

### Challenge · +30점

| 항목 | 배점 | 적용 기준 |
|---|---:|---|
| 후속 전략 실효성 | 20 | 캠페인 기획안이 근거 리드와 연결되고 바로 실행 가능 |
| 양식·기준 설계 | 10 | 캠페인 기획안까지 자동 생성되며, 이메일 양식, review_flag, 우선순위 판단 기준을 직접 설계 |

## 4. 프로젝트 자기점검 체크리스트

- [ ] 문제를 48시간 골든타임과 직책별 개인화 문제로 재정의했다.
- [ ] `company-info.md`와 `industry-news.md`를 필수 컨텍스트로 반영했다.
- [ ] Basic 산출물의 필수 컬럼과 본문 조건을 충족했다.
- [ ] Standard 파이프라인 흐름이 새 CSV에도 적용 가능하다.
- [ ] Challenge 캠페인 기획안이 파이프라인에서 자동 생성되고, 샘플 1~2건이 실제 리드 근거와 업계 맥락을 포함한다.
- [ ] 검수 리포트로 평가 항목별 통과 여부를 확인했다.
- [ ] 실제 이메일 발송, CRM 연동, 다국어, 리드 스코어링, API 키·외부 발송 연동을 하지 않았다.
