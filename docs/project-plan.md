# 프로젝트 기획 · Sparta_AX_Project

## 문제 정의
- (주제 미정) 해커톤에서 한 번 끝까지 풀 문제를 정한 뒤 여기에 적는다.
- 핵심 질문: "이 작업을 다음에도 똑같이 하려면 무엇이 필요한가?" → 그 답을 스킬로 남긴다.

## 목표와 성공 기준
- 작업을 `/스킬명` 한 줄로 **재현 가능한 도구**로 남긴다.
- 성공 기준: 동료가 `/스킬명`만 입력해도 같은 품질의 결과가 재현된다.

## 범위
- 포함: 스킬 한 개(SKILL.md) + 한 번 실행한 결과(입력 → 출력)
- 제외: 대규모 앱/서비스, 배포 인프라 (해커톤 제출 범위 밖)

## 주요 사용자와 사용 시나리오
- 누가: 사내 동료
- 언제/어떻게: 같은 작업이 필요할 때 `/스킬명 <작업 내용>` 또는 "○○ 스킬로 ~ 해줘"

## 산출물
- `.claude/skills/<스킬명>/SKILL.md`
- `out/demo/`에 입력 → 출력 1회 실행 결과
- 시연 1~2분 (`/스킬명` 호출)

## 일정·마일스톤
- 마감: `확인 필요`
- 주요 단계: 주제 확정 → 기준계약 작성 → SKILL.md 작성 → 시연 검증 → 제출

## 리스크와 대응
- 주제 미확정: 확정 즉시 기준계약부터 고정
- 트리거 키워드 모호 → 스킬이 안 불림: 실제로 말할 표현을 description에 그대로 넣음
- 되돌리기 어려운 작업 포함 시: 자동 실행 차단, `/스킬명` 수동 호출만 허용
