# [참고] 내 작업을 Skill로 만들어 제출하기 (해커톤 노션 공지 보관본)

> 출처: 스파르타 AX 해커톤 노션 공지 / 확인일: 2026-06-30 / 적용 범위: 제출물 형식·구조 기준

해커톤에서 만든 작업은 'Skill'로 만들어 제출한다. 한 번 만들어 두면 다음부터 `/스킬명` 한 줄로 같은 결과를 다시 뽑을 수 있고, 사내에서 누구나 그대로 가져다 쓸 수 있다. "한 번 해봤다"가 아니라 "재사용 가능한 도구로 남겼다"가 제출물의 핵심.

## 1. 왜 Skill로 제출하나
- 매번 같은 품질로 재현 — 운에 기대지 않음
- 동료가 `/스킬명`만 입력해도 바로 사용 — 사내 전파
- 결과 파일 하나보다 임팩트가 큼 — 작업을 '도구'로 남김

## 2. 작업을 Skill로 만들기
문제를 한 번 끝까지 풀었다면 그 작업을 그대로 Skill로 저장한다. 예시 지시:
```
방금 한 작업을 Skill로 만들어줘.
- .claude/skills/이름/SKILL.md 로 저장
- description에 내가 실제로 말할 트리거 키워드를 넣고
- 절차는 체크리스트로, 입출력 예시 1~2개도 넣어줘
```
> TIP: 안 떠오르면 "이 작업을 다음에도 똑같이 하려면 뭐가 필요해?"라고 먼저 물어보고, 그 답을 Skill로 저장.

## 3. 만든 Skill 쓰기
- `/스킬명` + 작업 내용 — 예: `/홍보글 여름 코딩캠프, 7월 14~25일, 조기등록 20% 할인`
- 자연어 언급 — 예: "홍보글 스킬로 이 행사 글 만들어줘"
- 헷갈리면 "지금 쓸 수 있는 skill 목록 보여줘"

## 4. 제출
- `SKILL.md` 파일 + 한 번 실행한 결과(입력 → 출력)를 함께 제출
- 시연은 `/스킬명`을 불러 결과가 나오는 1~2분이면 충분
> TIP: 메일 보내기·파일 삭제처럼 되돌리기 어려운 작업이 든 Skill은 자동 실행을 막고 `/이름`으로만 실행되게 해두면 안전.
