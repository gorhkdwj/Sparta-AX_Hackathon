> - **CTA** (Call To Action): 이메일 수신자가 취하길 바라는 단일 다음 행동

## ① 기업 소개

키즐링(주)은 알파세대 어린이를 위한 **안전한 어린이 전용 숏폼 챌린지 플랫폼**과 **학습경험플랫폼(LXP)**을 운영하는 에듀테크 스타트업입니다. 아이들이 영어스피치·줄넘기·그림·음악 챌린지에 숏폼 영상으로 참여하고, AI가 콘텐츠를 추천·평가·필터링해 안전하게 재능을 발굴·기록하게 합니다(유해 콘텐츠 차단율 99%). **CES 2026 혁신상**, **구글 창구 7기**, **중기부 디딤돌 R&D**에 선정된 성장형 스타트업입니다. B2B 핵심 제품은 **키즐콘(Kizlcon)** — 학교·학원·교육기관을 위한 온라인 대회 접수·결제·심사·결과발표 **원스톱 SaaS**입니다.

## ② 현재 페인포인트

키즐링 영업팀은 **에듀플러스위크** 등 교육 박람회에서 수십 명의 명함과 상담 메모를 수집합니다. 그런데 후속 이메일은 **담당자 1인이 전부 수작업**으로 작성합니다.

**[가상 시나리오]** 박람회 하나가 끝나면 리드가 50명씩 쌓입니다. 후속 이메일을 리드 1명당 10~15분씩 쓰면 50명이면 **8시간 이상** — 1인이 감당 불가입니다. 게다가 원장은 "원생 효과", 팀장은 "도입이 쉬운가", 사업개발은 "수익 모델"을 묻는데, **같은 톤의 이메일은 누구에게도 꽂히지 않습니다.** 부스에서 나눈 대화가 이메일에 반영되지 않으면 "우리를 기억 못 하는 회사"라는 인상을 주고, 박람회 후 **48시간 골든타임**이 지나면 리드의 관심이 급격히 식어 경쟁사에 기회를 빼앗깁니다.

## ③ 해결 목표

Claude Code와 대화하며 박람회 리드 데이터를 다루는 과제입니다. 아래 **3단계(Basic → Standard → Challenge)** 중 본인 수준에 맞는 목표를 골라 도전하세요. **위 단계는 아래 단계를 포함하는 누적형**이며, 더 높은 단계일수록 "1회성 초안"이 아니라 "매 박람회마다 돌아가는 자동화"로 발전합니다. 별도 API 키나 추가 비용 없이 Claude Code 대화 안에서 완료할 수 있습니다.

> 🟢 **Basic — 직책별 개인화 후속 이메일 초안 일괄 생성**

> 제공된 `leads_sample.csv`(50건)를 분석해 각 리드에 대해 **직책별(원장/팀장/사업개발) 소구점이 다른 개인화 이메일 초안**을 생성합니다. 각 이메일은 ①맞춤 인사 ②부스 상담내용 언급 ③직책 소구점→제품(키즐콘 기능) 연결 ④명확한 CTA의 4요소를 포함하고, 상담 메모가 비었거나 모호한 리드·이메일 없는 리드는 `review_flag`로 예외 처리합니다. 제출: `email_drafts.csv`/`.xlsx` + 노션/구글 시트 공개 링크.

> *핵심 역량: 데이터 정제 + 직책 분기 + 개인화 카피라이팅 (콘텐츠 마케터 수준)*

> 🟡 **Standard — 새 리드에도 도는 재현 파이프라인**

> Basic을 1회성 작업이 아니라 **재사용 가능한 파이프라인**으로 만듭니다. 새 리드 CSV를 넣으면 `정제 → 직책 분기 → 초안 생성 → review_flag`까지 같은 품질로 자동 생성되도록 Claude Code 슬래시 커맨드·스킬 또는 스크립트로 구성합니다. 직책별 소구점·CTA·우선순위별 강도는 **본인이 직접 설계**하고 마케팅·영업 관점 근거를 설명합니다.

> *핵심 역량: 워크플로 설계 + 재현성 + 자동화 (마케팅 PM 수준)*

> 🔴 **Challenge — 파이프라인 + 후속 캠페인 기획안 생성**

> Standard에 더해, 리드 데이터에서 발견한 패턴을 기반으로 **후속 캠페인 기획안까지 자동 생성**합니다. 각 기획안은 `세그먼트 정의 / 근거 리드(건수·인용) / 후속 시퀀스 / 우선순위`를 포함합니다. 이메일 양식·review_flag 기준·우선순위 판단 기준도 본인이 설계합니다. 즉, 리드 수집·초안 생성부터 데이터 기반 후속 전략 도출까지 한 번에 이어지는 봇을 만드는 것이 목표입니다.

> *핵심 역량: 인사이트 → 액션 전환 + 멀티스텝 자동화 (콘텐츠 전략가 수준)*

**기본 작업 흐름 (Basic 기준):**

```jsx
leads_sample.csv 파일을 Claude Code에 올리기
        ↓
Claude Code와 대화로 정제·직책 분기·개인화 초안·review_flag 도출
        ↓
email_drafts.csv/.xlsx 로 저장 → 노션/구글 시트 공개 링크 제출
```

> Standard·Challenge는 위 흐름을 **매 박람회마다 자동 반복되는 파이프라인/봇으로 확장**하는 것이 목표입니다.

## ④ 제공 더미 데이터

- **파일명**: `data/leads_sample.csv`
- **행수**: 50행 (원장 20명 / 팀장 18명 / 사업개발 12명, 중복 1건 포함)

| 컬럼명                 | 설명                                                     |
| ---------------------- | -------------------------------------------------------- |
| `lead_id`            | 리드 고유번호 (L001~L050)                                |
| `name`               | 담당자 이름                                              |
| `organization`       | 소속 기관·기업명                                        |
| `org_type`           | 학원 / 학교 / 에듀테크기업 / 공공기관                    |
| `title`              | 원장 / 팀장 / 사업개발                                   |
| `email`              | 이메일 주소 (일부 결측)                                  |
| `consultation_note`  | 부스 상담 메모 (일부 결측·모호)                         |
| `interest_feature`   | 대회운영SaaS / 콘텐츠안전필터 / 파트너십                 |
| `follow_up_priority` | 상 / 중 / 하 (CTA 강도 조절에 활용, 직접 점수 산출 금지) |

> ⚠️ **데이터 품질 주의** — 실무 데이터처럼 의도적으로 깔끔하지 않습니다. 정제·예외 처리가 필요합니다.

> - 이메일 결측: 1건 (이메일 없는 리드 → 발송 불가, 별도 후속 메모로 전환)

> - 상담 메모 결측: 2건 (빈 셀 또는 "상담 메모 없음")

> - 상담 메모 모호: 1건 (니즈 파악이 안 되는 메모)

> - 위 결측·모호 케이스는 `review_flag = Y` (사람이 보강 후 발송)

> - 완전 중복 리드: 1건 (lead_id만 다르고 내용 동일 → 한 건으로 처리)

## ⑤ 기대 Output

- **제출 형식**: `email_drafts.csv` 또는 `.xlsx` (컬럼: `lead_id, subject, body, review_flag`) — 노션 페이지 또는 구글 시트 공개 링크로 제출. Standard·Challenge는 파이프라인(슬래시 커맨드/스킬/스크립트)도 함께 제출합니다.

**🟢 Basic — 산출물 고정 (아래 항목 그대로 제출)**

- **포함 항목**:
  - **리드별 이메일 초안 테이블**: `lead_id` / `subject`(직책별 차별화) / `body`(인사→상담언급→제품연결→CTA, 250~400자) / `review_flag`(Y/N)
  - **직책별 차별화**: 원장·팀장·사업개발 3직책이 서로 다른 소구점·CTA·제목을 사용 (원장=무료 데모 신청 / 팀장=기술 도입 미팅 / 사업개발=파트너십 제안서 송부)
  - **개인화 4요소**: 각 이메일에 ①맞춤 인사 ②부스 상담내용 언급 ③직책 소구점→키즐콘 기능 연결 ④명확한 CTA 1개 포함
  - **예외 처리**: 상담 메모 결측·모호 리드는 `review_flag = Y`, 이메일 없는 리드는 발송 대신 별도 후속 메모로 전환

**🟡 Standard — 최소 요건 + 본인 설계**

- (필수) 재사용 가능한 파이프라인: 새 리드 CSV를 넣으면 `정제→직책 분기→초안 생성→review_flag`가 동일 품질로 자동 생성되는 슬래시 커맨드/스킬/스크립트 + 간단한 사용법
- (필수) 위 파이프라인으로 생성한 이메일 초안 샘플 1부
- (직접 기획) 직책별 소구점·CTA·우선순위별 강도를 "마케팅·영업 관점에서 무엇이 전환을 만드는가"를 기준으로 **본인이 설계** → 왜 그렇게 구성했는지 1~2줄 설명 포함

**🔴 Challenge — Standard + 캠페인 기획안, 양식도 본인 설계**

- (필수) Standard 산출물 전체
- (필수) 리드 데이터에서 도출한 후속 캠페인 기획안 샘플 1~2건 (세그먼트 정의 / 근거 리드 / 후속 시퀀스 / 우선순위)
- (직접 기획) 이메일 양식·review_flag 기준·우선순위 판단 기준을 **본인이 설계** → 판단 근거 설명 포함

> 💡 Standard·Challenge는 "정해진 양식 채우기"가 아니라 **어떤 Output이 현업에 유용한지 스스로 정의하는 것**까지 과제에 포함됩니다.
