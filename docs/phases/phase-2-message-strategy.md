# Phase 2 · 메시지 전략과 이메일 생성 규칙 설계

## 목표

키즐링 도메인 맥락과 업계 맥락을 바탕으로 직책별 소구점, 제목, CTA, 우선순위별 강도, review_flag 기준을 확정한다.

## 입력 자료

- `키즐링/context/company-info.md`
- `키즐링/context/industry-news.md`
- `.claude/skills/kizzling-message-strategy/SKILL.md`
- 제공 참고 자료: `키즐링/.claude/skills/insight.md`
- Phase 1 데이터 분석 요약
- `docs/requirements-contract.md`

## 수행 계획

1. 키즐링과 키즐콘의 핵심 기능을 이메일 생성 규칙에 연결한다.
2. 직책별 메시지 전략을 확정한다.
   - 원장: 원생 재능 발굴, 학부모 만족, 기관 브랜딩
   - 팀장: 도입 난이도, 운영 공수 절감, 기존 시스템 연동
   - 사업개발: 파트너십 수익 모델, 공동 마케팅, 기관 공급 계약
3. 직책별 CTA를 고정한다.
   - 원장: 무료 데모 신청
   - 팀장: 기술 도입 미팅
   - 사업개발: 파트너십 제안서 송부
4. 우선순위별 CTA 강도를 설계한다.
   - 상: 48시간 내 즉시 행동
   - 중: 일정 조율과 자료 안내
   - 하: 사례집·소개자료 기반 장기 너처링
5. 이메일 본문 구조를 고정한다.
   - 인사 → 상담언급 → 제품연결 → CTA
6. Standard 요구사항에 맞춰 마케팅·영업 관점 설계 근거를 1~2줄 이상 작성한다.
7. 주요 판단은 `키즐링/decisions.md` 또는 `Decisionlog.md`에 기록한다.

### 우선순위별 CTA 강도

| follow_up_priority | CTA 강도 | 작성 기준 |
|---|---|---|
| 상 | 강함 | 48시간 골든타임 내 이번 주 데모·미팅·제안서 송부처럼 즉시 행동을 제안 |
| 중 | 보통 | 관심을 유지하되 일정 조율과 자료 안내 중심으로 부담을 낮춤 |
| 하 | 약함 | 사례집·소개자료·체크리스트 전달 등 장기 너처링 중심으로 접근 |

`follow_up_priority`는 주어진 입력값으로만 사용하며 새 점수나 리드 스코어를 계산하지 않는다.

### 이메일 본문 구조

모든 발송 가능 이메일은 아래 순서를 따른다.

1. 맞춤 인사: `{name} {직책 호칭}` + 에듀플러스위크 만남 감사
2. 상담언급: `consultation_note`의 구체적 대화 1문장 반영
3. 제품연결: `title` 소구점과 `interest_feature` 키즐콘 기능 연결
4. CTA: 직책별 단일 CTA 1개

본문은 한국어 250~400자, 자연스러운 비즈니스 문체로 작성한다.

### review_flag 기준

| 조건 | review_flag | 본문 처리 |
|---|---|---|
| 상담 메모가 구체적이고 이메일이 있음 | N | 상담 내용을 구체적으로 언급 |
| 상담 메모 빈 셀 또는 `상담 메모 없음` | Y | 상담 내용을 일반화하고 사람 보강 후 발송 |
| 상담 메모가 모호해 니즈 파악 불가 | Y | 상담 내용을 일반화하고 사람 보강 후 발송 |
| 이메일 결측 | Y | 이메일 초안 대신 발송 불가 후속 메모로 전환 |
| 완전 중복 리드 | 별도 제외 기록 | 기준 lead_id만 사용하고 제외 lead_id를 기록 |

Phase 1 기준 `review_flag = Y` 후보는 L009, L012, L015, L033, L046이다. L050은 L039 중복 제외로 처리한다.

### 제목 생성 규칙

| title | 제목 템플릿 방향 |
|---|---|
| 원장 | `[키즐링] {name} 원장님, 우리 원 아이들의 재능을 보여주는 키즐콘 안내` |
| 팀장 | `[키즐링] {name} 팀장님, 기존 운영에 쉽게 붙는 대회 운영 자동화` |
| 사업개발 | `[키즐링] {name} 님, 키즐콘 공동 공급·파트너십 제안의 건` |

제목은 직책별 관심사가 드러나야 하며, “무료”, “할인” 같은 스팸 유발 단어를 남발하지 않는다. 단, 원장 CTA의 “무료 데모 신청”은 본문 CTA에서만 사용한다.

### Standard 설계 근거

- 직책별로 구매 동기가 다르기 때문에 같은 제품을 원장에게는 원생·학부모 효과, 팀장에게는 도입 난이도와 공수 절감, 사업개발에게는 수익 모델과 공동 공급 기회로 재구성해야 전환 가능성이 높다.
- 박람회 리드는 48시간 내 후속과 상담 내용 언급이 중요하므로, `follow_up_priority`가 높은 리드는 즉시 행동 CTA를 쓰고 모호한 상담 메모는 Human-in-the-loop 방식으로 `review_flag = Y`를 부여한다.
