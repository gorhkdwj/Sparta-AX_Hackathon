# 구현 계획 · Sparta_AX_Project (스킬 작성)

> 처음부터 전체를 달리지 않는다. 가장 작은 성공 단위부터. 각 단계는 완료 조건과 검증 방법을 명시한다.

## 단계 개요
| 단계 | 목표 | 완료 조건 | 검증 방법 | 의존/연결 |
|------|------|-----------|-----------|-----------|
| S1 | 해커톤 주제 확정 + 작업 1회 끝까지 풀기 | 결과물 1개 산출 | 결과 눈으로 확인 | — |
| S2 | 기준계약 고정(트리거/입력/출력/절차) | requirements-contract 채움 | 사용자 합의 | S1 |
| S3 | SKILL.md 작성 (skill-creator 활용) | description+체크리스트+예시 포함 | 구조 점검 | S2 |
| S4 | 시연: `/스킬명` 호출 → 입력→출력 재현 | out/demo에 결과 저장 | 예시와 일치 확인 | S3 |
| S5 | 제출 패키징 | SKILL.md + 실행결과 정리 | 제출 가이드 대조 | S4 |

## 단계 상세

### S3 · SKILL.md 작성
- 왜 필요한가: 작업을 재사용 가능한 도구로 고정
- 이전 단계와 연결: S2의 기준계약을 그대로 SKILL.md로 옮김
- 다음 단계로 전달: S4 시연 입력
- 완료 조건: `name`, `description`(트리거 키워드), 절차 체크리스트, 입출력 예시 1~2개 포함
- 검증 방법: `/스킬명` 호출이 의도대로 트리거되는지 + 예시 출력 재현
- 실패 시 중단점: 스킬이 안 불리면 description의 트리거 표현부터 수정
