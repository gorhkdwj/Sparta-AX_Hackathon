# 검증 계획 · Sparta_AX_Project (스킬)

스킬은 "실제로 불러서 입력 → 출력이 재현되는가"로 검증한다. 검증하지 못한 부분은 숨기지 않고 `미검증 범위`에 적는다.

## 검증 항목
- 트리거: `/스킬명` 및 자연어 호출로 스킬이 실제로 불리는가
- 입력 처리: 의도한 입력 형식을 받아 처리하는가
- 출력 재현: SKILL.md의 입출력 예시와 동일한 결과가 나오는가
- 문서 정합성: requirements-contract ↔ SKILL.md ↔ out/demo 결과 일치
- 제출물 구조: SKILL.md + 실행결과(입력→출력)가 제출 가이드 형식에 맞는가
- 안전장치: 되돌리기 어려운 작업의 자동 실행 차단 동작 확인 (해당 시)

## 검증 방법과 도구
- Claude Code에서 `/스킬명 <예시 입력>` 직접 실행 → 결과를 `out/demo/`에 저장
- "지금 쓸 수 있는 skill 목록 보여줘"로 스킬 등록 확인

## 미검증 범위
- (현재) 주제 미정으로 스킬 본문 미작성 → 전 항목 미검증. 주제 확정 후 진행.
