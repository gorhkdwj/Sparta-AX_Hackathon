# 키즐링 캠페인 기획안 생성

리드 데이터와 이메일 초안 산출물을 바탕으로 Challenge 수준의 캠페인 기획안을 만든다. 리드 근거 없이 캠페인을 지어내지 않는다.

## 입력

다음 자료를 사용한다.

- 리드 분석 handoff
- 메시지 전략 handoff
- 이메일 초안 패키지
- 키즐링 회사 컨텍스트와 업계 컨텍스트

## 세그먼트 규칙

다음 값을 조합해 세그먼트를 정의한다.

- `org_type`
- `title`
- `interest_feature`
- `follow_up_priority`
- 운영 순서에 필요할 경우 `review_flag` 상태

실제 리드 행으로 추적할 수 없는 세그먼트를 만들지 않는다.

## 캠페인 비교 요약 (기획안이 2건 이상일 때 개별 기획안 앞에 배치)

여러 캠페인을 한눈에 비교해 어디부터 착수할지 바로 판단할 수 있도록 표를 넣는다. 새로운 점수나 순위를 계산하지 않고, 이미 산출된 값(리드 수, `상` 비중, 검수 필요 건수)만 나란히 배치한다.

```text
| 캠페인 | 세그먼트 규모(실질) | 우선순위 `상` 비중 | 검수 필요 리드 | 핵심 리스크 |
|---|---|---|---|---|
| 1. [이름] | 12건 | 6/12 (50%) | 1건 | (한 줄 요약) |
| 2. [이름] | 12건(실질 11건) | 5/11 (45.5%) | 1건 | (한 줄 요약) |
```

## 캠페인 기획안 형식

기획안은 1~2건 생성한다. 각 기획안은 다음 형식을 따른다.

```text
## 캠페인 N. [이름]

세그먼트 정의:
근거 리드:
대표 상담 인용:
핵심 인사이트:
후속 시퀀스 (체크리스트, 각 항목에 담당 부서 표기):
- [ ] Day 0 (담당: OO팀) — 액션 내용
- [ ] Day 2~3 (담당: OO팀) — 액션 내용
- [ ] Day 7 (담당: OO팀) — 액션 내용
우선순위:
우선순위 근거:
리스크·차단요소: 무엇이 이 시퀀스 실행을 늦추거나 막을 수 있는지, 그리고 그걸 막기 위해 누가 언제까지 무엇을 확인해야 하는지
담당자 액션:
이메일/review_flag/우선순위 설계 근거:
```

## 후속 시퀀스 패턴

실무적인 B2B 후속 타이밍을 사용하고, 각 단계마다 실행 담당 부서를 명시한다. 담당 배정은 CTA 성격을 기준으로 정한다.

- Day 0 (담당: 마케팅팀): 개인화 후속 이메일 발송, 또는 이메일 결측 리드는 연락처 확인 착수
- Day 2~3 (담당: 기술 도입 미팅 CTA 세그먼트는 세일즈엔지니어링/솔루션팀, 파트너십 제안서 CTA 세그먼트는 사업개발팀): 사례 자료, 연동 체크리스트, 파트너십 1페이지 제안서 등 전달
- Day 7 (담당: 영업팀 또는 사업개발팀): 미팅, 데모, 제안 후속

`review_flag = Y`가 포함된 그룹은 발송 전에 사람 검수 또는 연락처 확인 단계를 넣고, 이 확인 작업의 담당과 기한을 리스크·차단요소에 명시한다.

## 리스크·차단요소 작성 기준

- 무엇이 지연·중단을 일으킬 수 있는지 구체적 리드(`lead_id`)를 근거로 든다(예: review_flag 리드의 확인 지연이 48시간 골든타임을 넘기는 경우).
- 그 리스크를 낮추기 위해 어느 부서가 언제까지 무엇을 확인해야 하는지까지 적는다.
- 새로운 위험 점수나 확률 수치를 만들지 않는다. 정성적으로 서술한다.

## 우선순위 규칙

제공된 `follow_up_priority`를 사용하고, 새 점수 체계를 만들지 않는다.

우선순위 근거에는 다음을 활용할 수 있다.

- 세그먼트 리드 수
- `상` 우선순위 리드 비중
- 48시간 골든타임
- 매출 또는 파트너십 가능성
- 사람 검수가 필요한 데이터 품질 리스크

## 전달

캠페인 기획안을 `kizzling-visualize-campaign`으로 넘긴다. 시각화가 필요 없으면 `kizzling-review-submit`으로 바로 넘긴다.
